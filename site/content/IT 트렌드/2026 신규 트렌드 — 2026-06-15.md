## WebGPU × WASI

### 2026-06-15 기준 트렌드 개요

WebGPU는 브라우저 내 GPU 연산 표준으로 Chrome·Firefox·Safari 모두 정식 지원 중이며, 2026년 들어 **wasi-webgpu** 제안이 WebAssembly System Interface 레벨로 확장되면서 서버·엣지 런타임까지 영향권이 넓어졌다. Hacker News에서 2026-06-09(wasi-gfx 미래), 2026-06-12(wasi-webgpu GitHub), 2026-05-26(브라우저 내 RL 에이전트) 등 최근 한 달간 6건 이상 클러스터가 형성됐다. llama.cpp의 WebGPU 백엔드 지원(2026-05-21)은 **브라우저 내 LLM 추론** 가능성을 현실화하고 있어 Edge AI와의 교차점이 주목된다. 성숙도는 '실험적 채택 단계'이나 표준화 속도가 빠르다.

> [!tip] 핵심 포인트
> wasi-webgpu가 표준화되면 Cloudflare Workers·Wasmtime 등 서버 WASM 런타임에서도 GPU를 직접 활용할 수 있게 된다.

### 핵심 기술 스택

- **WebGPU API** — W3C 표준, `navigator.gpu` 진입점
- **wgpu** (Rust) — 크로스 플랫폼 WebGPU 구현체
- **wasi-webgpu** — WebAssembly/wasi-webgpu (GitHub proposal)
- **llama.cpp WebGPU 백엔드** — 브라우저 내 LLM 추론
- **three.js / Babylon.js** — WebGPU 렌더 파이프라인 지원
- **DawnJS** — Chrome의 WebGPU 구현체

### 추천 학습 자료

## DuckDB 임베디드 OLAP

### 2026-06-15 기준 트렌드 개요

DuckDB는 "SQLite의 OLAP 버전"으로 불리며, 2026년 6월 현재 데이터 분석·BI·파이프라인 전 계층에서 폭발적으로 채택되고 있다. Hacker News에서 2026-06-14(DuckDB+S3+Plotly), 2026-06-13(Why DuckDB), 2026-06-11(DuckDB WASM), 2026-06-06(DuckDB vs SQLite), 2026-06-05(Ladybug: DuckDB of Graph DB) 등 최근 2주 동안만 5건 클러스터가 형성됐다. **DuckDB WASM** 을 통해 브라우저에서 쿼리를 직접 실행하는 사례도 늘고 있으며, MotherDuck(클라우드 DuckDB)이 SaaS로 성숙해지면서 프로덕션 채택이 가속화됐다. SQLite 대비 집계 성능이 수십 배 이상이어서 Python/R 분석 환경의 기본 DB로 빠르게 자리잡고 있다.

> [!tip] 핵심 포인트
> `pip install duckdb` 하나로 시작 가능. Parquet·CSV·JSON·Arrow를 파일 시스템 수준에서 직접 쿼리할 수 있어 별도 ETL 없이 분석 가능.

> [!warning] 주의/한계
> 동시 쓰기(multi-writer)는 지원하지 않는다. 단일 프로세스 쓰기 + 다중 읽기 모델에 적합하며, OLTP 워크로드에는 부적합.

### 핵심 기술 스택

- **DuckDB** (Python/R/Node/Java/Rust 클라이언트)
- **DuckDB WASM** — 브라우저 내 OLAP
- **MotherDuck** — 클라우드 관리형 DuckDB
- **Apache Arrow / Parquet** — 네이티브 파일 포맷 지원
- **dbt-duckdb** — dbt 어댑터로 로컬 데이터 변환 파이프라인
- **Plotly / Observable** — DuckDB WASM 기반 시각화

### 추천 학습 자료

- [DuckDB WASM (GitHub)](https://github.com/duckdb/duckdb-wasm)
- [DuckDB vs. SQLite (MotherDuck)](https://motherduck.com/learn/duckdb-vs-sqlite-databases/)

- [DuckDB 공식 문서](https://duckdb.org/docs/)

### 적합한 역할

- 백엔드 개발자 (내장 분석 DB 필요 시)
- ML 엔지니어 (피처 스토어 경량 대안)

## Local-First 데이터 (CRDT)

### 2026-06-15 기준 트렌드 개요

"Local-First Software" 패러다임은 오프라인 우선 동작·사용자 데이터 소유권을 핵심으로 하며, 2026년 들어 **Rust + SQLite + CRDT** 조합으로 구현된 Show HN 프로젝트가 급증하고 있다. Hacker News 최근 1개월 내 OpenBrief(2026-05-25), Mnemo(2026-06-03), Cortex(2026-06-13), Viddcut(2026-06-13), GlyphX(2026-06-14) 등 6건 이상이 'local-first' 키워드로 등장했다. 특히 **Rust + MCP(Model Context Protocol)** 와 결합한 AI 메모리 레이어가 새로운 패턴으로 부상하고 있다. Automerge 2.x·Yjs는 성숙 단계로 프로덕션 적용 사례가 늘고 있으며, Liveblocks·PartyKit 등 관리형 CRDT 서비스도 확산 중.

> [!tip] 핵심 포인트
> CRDT(Conflict-free Replicated Data Type)는 네트워크 없이도 여러 노드가 독립적으로 수정 후 병합 가능. 실시간 협업(Notion·Figma류)과 오프라인 우선 앱 모두에 적합.

> [!warning] 주의/한계
> CRDT 선택(Automerge vs Yjs)이 나중에 바꾸기 어렵다. 데이터 구조 설계 초기에 충분히 검토해야 하며, 삭제 처리(tombstone 관리)에 주의.

### 핵심 기술 스택

- **Automerge 2.x** (Rust/JS) — 문서형 CRDT
- **Yjs** — 텍스트·맵 CRDT, Tiptap·ProseMirror 통합
- **SQLite + CR-SQLite** — CRDT 확장을 붙인 SQLite
- **Liveblocks** — 관리형 실시간 협업 백엔드
- **PartyKit** — 엣지 기반 실시간 멀티플레이어
- **ElectricSQL** — 로컬 Postgres 동기화

### 추천 학습 자료

- [Local-First Software (Ink & Switch 논문)](https://www.inkandswitch.com/local-first/)
- [Automerge 공식 문서](https://automerge.org/)
- [Yjs 공식 문서](https://docs.yjs.dev/)
- [CR-SQLite GitHub](https://github.com/vlcn-io/cr-sqlite)
- [Mnemo — local-first AI memory (Rust/SQLite)](https://github.com/zaydmulani09/mnemo)
- [Cortex — local-first AI agent memory (Rust/MCP)](https://github.com/gambletan/cortex)

### 적합한 역할

- 풀스택 개발자 (협업 SaaS 개발)
- 모바일/데스크톱 앱 개발자 (오프라인 우선 UX)
- AI 에이전트 개발자 (로컬 메모리 레이어 구축)

## Zig 시스템 프로그래밍 언어

### 2026-06-15 기준 트렌드 개요

Zig는 C의 직접적인 대안으로 포지셔닝되며 2026년 현재 **시스템 프로그래밍·드라이버·임베디드** 영역에서 꾸준한 채택 신호가 포착된다. Hacker News에서 2026-06-06 'Zig로 작성된 Vulkan 드라이버 Ape' 등장은 실제 시스템 레벨 프로젝트에의 적용을 보여준다. Rux(2026-05-31), Ü(2026-06-04) 등 Zig 영향을 받은 신규 시스템 언어들도 연이어 등장하고 있어 생태계 관심이 고조됨을 반영한다. Zig 0.14 릴리스 이후 빌드 시스템 안정화와 패키지 매니저 통합이 이뤄졌으며, C 라이브러리 interop 없는 컴파일 환경이 크로스 컴파일 사례를 증가시키고 있다. 다만 1.0 이전(pre-stable)으로 언어 변경이 잦다.

> [!tip] 핵심 포인트
> `zig cc`를 C 컴파일러 드롭인 대체로 쓰는 것만으로도 크로스 컴파일·정적 링크 문제를 해결할 수 있다. Zig를 도구로 먼저 경험하는 게 진입 장벽을 낮추는 지름길.

> [!warning] 주의/한계
> 아직 1.0 미만(pre-stable). API·언어 문법이 버전 간 변경될 수 있어 장기 프로덕션 적용은 신중해야 한다. Zig 패키지 에코시스템은 Rust/Go 대비 작다.

- **Ape** — Zig로 작성된 Vulkan 드라이버 (시스템 레벨 적용 사례)

## Tauri 네이티브 데스크톱

### 2026-06-15 기준 트렌드 개요

Tauri는 Electron의 경량 대안으로 Rust 백엔드 + 웹 프론트엔드 조합으로 데스크톱 앱을 빌드한다. 2026-06-07 Hacker News에 'Rune — Tauri/Rust 마크다운 에디터'가 게시되는 등 커뮤니티 Show HN 제출이 이어지고 있다. Tauri 2.0이 정식 출시되면서 iOS·Android 모바일 지원까지 추가됐으며, Electron 대비 번들 크기 10배 감소·메모리 절감이 핵심 채택 이유다. Wails(Go 백엔드)와 함께 '비 Electron 데스크톱 프레임워크' 양대 산맥으로 거론된다. 성숙도는 Tauri 2.0 기준 프로덕션 적용 가능 수준으로 평가된다.

> [!tip] 핵심 포인트
> Tauri 2.0부터 **모바일(iOS/Android) 타겟**도 동일 코드베이스로 지원. 프론트엔드는 기존 React/Vue/Svelte 그대로 사용 가능해 웹 개발자 진입 장벽이 낮다.

> [!warning] 주의/한계
> Rust 백엔드 커스텀 명령어 작성 시 Rust 기초 지식 필요. WebView 기반이라 플랫폼별 렌더링 차이가 발생할 수 있고, 복잡한 GPU/3D 작업은 Electron보다 불편할 수 있다.

### 핵심 기술 스택

- **Tauri 2.0** — tauri.app
- **Rust** — Tauri 코어 및 백엔드 명령
- **Vite + React/Vue/Svelte** — 프론트엔드 레이어
- **Wails** (Go 기반 대안) — wails.io
- **tauri-plugin-store** — 로컬 영속 스토리지
- **tauri-plugin-updater** — 자동 업데이트
