기존 Vault 노트([[2026 신규 트렌드 — 2026-06-15]])는 AI 에이전트 오케스트레이션, DevOps 자동화, 파이프라인 보안 등 이미 메인스트림 진입 직전 단계의 기술을 정리했다. 오늘 2026-06-16 기준으로는 **Hacker News의 지난 30일 Show HN·토론 빈도**와 **Microsoft AI Diffusion Report Q1 2026** 신호를 교차 검토해, 아직 국내 개발자 커뮤니티에서 충분히 조명받지 못한 5개 카테고리를 선정했다. 각 카테고리는 단순 실험 단계를 넘어 프로덕션 채택 사례가 HN에 등장하기 시작한 '초기 메인스트림 진입' 시점에 해당한다.

## DuckDB · 임베디드 OLAP

### 2026-06-16 기준 트렌드 개요
DuckDB는 2026년 6월 5~14일 사이 HN에서만 5건 이상의 독립 토론 스레드가 생성될 만큼 꾸준한 커뮤니티 주목을 받고 있다. "DuckDB vs. SQLite"(2026-06-06), "Why DuckDB"(2026-06-13), "DuckDB WASM"(2026-06-11), "DuckDB + S3 + Plotly Studio"(2026-06-14) 등이 최근 30일 내 HN 상위권에 올랐다. 임베디드 OLAP 카테고리에서는 SlothDB(C++20 기반, ClickBench에서 DuckDB 초과)가 도전장을 냈고, Ladybug("DuckDB of Graph Databases")처럼 DuckDB 패러다임을 그래프 DB에 적용하는 파생 프로젝트도 등장했다. 성숙도: **프로덕션 안정(v1.x)**, 채택률 급등 구간.

> [!tip] 핵심 강점
> 별도 서버 없이 Python/Node/WASM 한 줄 import로 수백 GB 파케이·Parquet·S3 직접 쿼리 가능. 데이터 파이프라인·ML 피처 엔지니어링·BI 프로토타이핑에서 Pandas 대체재로 빠르게 자리 잡는 중.

### 핵심 기술 스택
- **DuckDB** 1.x (Python/Node.js/WASM/Rust 바인딩)
- **MotherDuck** — 서버리스 DuckDB 클라우드 레이어
- **dbt-duckdb** — 로컬 ELT 파이프라인
- **Apache Parquet / Iceberg** — 저장 포맷
- **duckdb-wasm** — 브라우저·엣지 환경 직접 쿼리

### 추천 학습 자료
- [Why DuckDB (공식)](https://duckdb.org/why_duckdb)
- [DuckDB WASM GitHub](https://github.com/duckdb/duckdb-wasm)
- [DuckDB vs SQLite — MotherDuck](https://motherduck.com/learn/duckdb-vs-sqlite-databases/)
- [DuckDB + S3 + Plotly Studio](https://chris-parmer.com/duckdb-s3-and-plotly/)

### 적합한 역할
데이터 엔지니어, ML 엔지니어, 백엔드 개발자(분석 쿼리 내재화), BI 개발자

## Local-First 소프트웨어 (CRDT · Rust · WASM)

### 2026-06-16 기준 트렌드 개요
2026년 6월 3~14일 HN Show HN에서 Local-First 키워드를 단 프로젝트가 연속 6건 이상 등장했다. Mnemo(Rust+SQLite, LLM 메모리, 2026-06-03), Cortex(Rust+MCP 암호화 메모리, 2026-06-13), Viddcut(Rust/WASM 비디오 에디터, 2026-06-13), GlyphX(오프라인 LaTeX, 2026-06-14) 등이 대표적이다. 공통 패턴은 **Rust 기반 로컬 스토리지 + WASM 크로스플랫폼 + 오프라인 우선** 아키텍처이다. CRDT(Automerge, Yjs)를 활용한 실시간 동기화도 병행 성장 중. 성숙도: **얼리 어답터→초기 메인스트림 전환**.

> [!tip] 핵심 강점
> 클라우드 의존을 제거해 개인정보 보호·오프라인 동작·지연 0ms UX를 동시에 달성. AI 에이전트 메모리 레이어(Mnemo, Cortex)와 결합되며 엣지 AI와 시너지.

> [!warning] 주의
> CRDT 병합 로직 디버깅이 복잡하고, 대용량 동시 편집 시 메모리 오버헤드가 발생할 수 있다. 팀 협업 제품엔 충분한 충돌 해소 UX 설계가 필요.

### 핵심 기술 스택
- **Automerge** / **Yjs** — CRDT 동기화 라이브러리
- **Rust + SQLite(rusqlite)** — 로컬 영속성
- **WebAssembly(WASM)** — 브라우저·크로스플랫폼 배포
- **libp2p / WebRTC** — P2P 동기화 트랜스포트
- **electric-sql / PowerSync** — 서버-클라이언트 CRDT 싱크

### 추천 학습 자료

- [Automerge 공식 문서](https://automerge.org/docs/)
- [Yjs 공식 문서](https://docs.yjs.dev/)
- [Show HN: Mnemo](https://github.com/zaydmulani09/mnemo)
- [Show HN: Cortex (Rust, MCP)](https://github.com/gambletan/cortex)

### 적합한 역할
프론트엔드 개발자, 풀스택 개발자, AI 에이전트 메모리 아키텍트, 오프라인 퍼스트 모바일 개발자

## MCP (Model Context Protocol) · AI 에이전트 표준 인터페이스

### 2026-06-16 기준 트렌드 개요
Anthropic이 제안한 MCP(Model Context Protocol)가 2026년 6월 AI 에이전트 툴링의 사실상 표준으로 부상 중이다. HN에서 6월 8~15일 사이 Web Speed(MCP 웹맵 레지스트리, 2026-06-08), AgentSploit(MCP 서버 침투 테스트 도구, 2026-06-09), Cortex(MCP 메모리, 2026-06-13), WordPress-MCP(2026-06-15) 등 다양한 생태계 프로젝트가 하루가 멀다 하고 등장하고 있다. 보안 특화 툴(AgentSploit — "Burp Suite for MCP")이 별도 등장한 것은 MCP 서버가 실제 프로덕션에 배포되기 시작했다는 신호다. 성숙도: **표준 제정 초기·생태계 폭발 단계**.

> [!tip] 핵심 강점
> LLM 모델을 교체하더라도 동일한 MCP 서버 인터페이스 재사용 가능. Claude, GPT-4o, Gemini 모두 MCP 클라이언트 지원 발표 — 벤더 락인 탈피.

> [!warning] 주의
> MCP 서버 자체가 새로운 공격 표면이 됨. AgentSploit 등 전용 침투 테스트 도구가 이미 나온 만큼, MCP 서버 배포 시 인증·샌드박싱 필수.

### 핵심 기술 스택
- **MCP SDK** (Python / TypeScript / Rust)
- **MCP Inspector** — 로컬 디버깅
- **MCP Registry** — 서버 디스커버리
- **AgentSploit** — MCP 보안 감사
- **Cortex / Mnemo** — MCP 기반 에이전트 메모리

### 추천 학습 자료
- [MCP 공식 스펙 (modelcontextprotocol.io)](https://modelcontextprotocol.io/)
- [MCP Python SDK](https://github.com/modelcontextprotocol/python-sdk)
- [Show HN: AgentSploit — Burp Suite for MCP](https://github.com/agentsploit/agentsploit)
- [Show HN: Web Speed — MCP 웹맵 레지스트리](https://www.getwebspeed.io/)

### 적합한 역할
AI 엔지니어, 백엔드 개발자(에이전트 툴 통합), DevSecOps(MCP 서버 보안), 플랫폼 엔지니어

## Tauri · Rust 기반 네이티브 데스크톱

### 2026-06-16 기준 트렌드 개요
Tauri(Rust + 시스템 WebView)가 Electron 대체재로 실무 전환 사례가 쏟아지고 있다. "Five months after switching Fluxzy from Electron to Tauri"(2026-05-19), "I Built the Same App with Five GUI Frameworks: Tauri·Slint·Egui·Dioxus·Flutter"(2026-06-01), Tauri 기반 음악 스위트(2026-06-05), Markdown 에디터(2026-06-07) 등 30일 내 HN 4건이 모두 실제 제품 전환 또는 Show HN이다. 번들 크기 10MB 미만·RAM 40MB 수준이라는 벤치마크가 Electron(200MB+) 대비 공유되며 채택을 가속. 성숙도: **v2 안정 릴리스, 프로덕션 채택 증가**.

> [!tip] 핵심 강점
> Rust 백엔드 + 프론트엔드 자유 선택(React·Vue·Svelte) 조합. Electron 대비 메모리·번들 크기 10배 이상 절감. 크로스플랫폼(Windows·macOS·Linux·Android·iOS 베타).

### 핵심 기술 스택
- **Tauri v2** (Rust 코어)
- **tauri-plugin-*** — 파일시스템·알림·업데이트 플러그인
- **Vite + React/Svelte** — 프론트엔드 번들
- **Slint / Dioxus / egui** — Rust 네이티브 UI 대안
- **tauri-action** — GitHub Actions CI/CD

## WebGPU · 브라우저 ML 추론

### 2026-06-16 기준 트렌드 개요
WebGPU가 Chrome·Firefox·Safari 주요 버전에서 안정 지원되면서, 브라우저에서 직접 ML 추론을 실행하는 프로젝트가 등장하기 시작했다. "Show HN: Run RL agents in the browser with WebGPU"(agenlus.com, 2026-05-26)는 강화학습 에이전트를 서버 없이 브라우저에서 실행한 사례로 HN 커뮤니티의 주목을 받았다. DuckDB-WASM과 결합해 데이터 처리→모델 추론을 클라이언트 사이드에서 완결하는 아키텍처가 Local-First 트렌드와 맞물려 성장 중이다. 성숙도: **표준 안정화 완료, 에코시스템 초기 성장**.

> [!tip] 핵심 강점
> GPU 컴퓨트 셰이더를 브라우저에서 직접 호출 → 서버 비용 0, 지연 최소화, 프라이버시 보호. Transformers.js v3·ONNX Runtime Web과 결합 시 LLM 경량 추론도 가능.

> [!warning] 주의
> 아직 모바일 Safari 지원이 불완전하고, 구형 GPU에서는 성능 편차가 크다. WebGPU API가 WebGL보다 저수준이라 학습 곡선 존재.

### 핵심 기술 스택
- **WebGPU API** (WGSL 셰이더 언어)
- **Transformers.js v3** — 브라우저 HuggingFace 모델 추론
- **ONNX Runtime Web** — 크로스 프레임워크 모델 실행
- **wgpu** (Rust) — 네이티브+웹 통합 GPU 추상화
- **TensorFlow.js WebGPU backend**

### 추천 학습 자료

- [Transformers.js 공식](https://huggingface.co/docs/transformers.js/)
- [Show HN: Run RL agents in the browser with WebGPU](https://agenlus.com/)
- [wgpu (Rust WebGPU)](https://wgpu.rs/)

### 적합한 역할
프론트엔드 엔지니어(ML 통합), 그래픽/게임 개발자, ML 엔지니어(엣지 추론), WebXR 개발자
